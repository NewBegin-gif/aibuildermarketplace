🛠️ Phase I: The Pre-Production Framework (The 30% Rule)

Most founders jump straight into the software, treating it like a video editor. That is the single biggest mistake. A great video starts on a document, not in a timeline. We spend 30% of our time planning to ensure the remaining 70% of the time is focused purely on execution. Following this structured approach is non-negotiable for generating genuinely effective training modules.

1. Defining the Single Key Outcome (SKO)

Before opening InVideo, define the Single Key Outcome. Do not list features; list transformations. Is the goal to get the user to "successfully generate a weekly report"? Or is the goal to "understand the difference between credits and licenses"? Your tutorial content must drive toward a measurable user action. This focused approach is key to strong content repurposing strategies.

2. Scripting for Breathability (The 7th Grade Rule)

Avoid technical jargon unless absolutely necessary, and if you use it, define it immediately. Draft your script in a tool like Notion or Google Docs. Structure the script using the 'tell-show-tell' methodology:

  1. Tell: State the problem (e.g., "Manually consolidating data is slow and error-prone.").
  2. Show: Walk through the solution using screen recordings and animations.
  3. Tell: Reiterate the benefit and call to action (e.g., "Our automated dashboard saves you 6 hours per week.").

Keep speaking time natural. Reading a robotic script aloud reveals pacing issues; practice reading the script out loud to ensure it sounds like an experienced founder talking to a teammate, not an AI reading a manual.

📊 Phase II: Deep Implementation Guide – Step-by-Step InVideo Workflow

This is the mechanism for executing your blueprint. We’re assuming the script is finalized, and the screen recordings (using tools like Loom or Camtasia) are ready.

Step 1: Project Structure and Template Selection

Goal: Set the tone of authority and professionalism.
Action: Select a template category within InVideo that closely mirrors corporate training or explainers. Do not choose a visually distracting, energetic template designed for TikTok dances. Look for clean lines, data visualization elements, and ample whitespace.
Pro Tip: Many founders overlook the power of 'Brand Kit' functionality. Load your exact brand fonts (e.g., Montserrat Bold, IBM Plex Sans) and corporate HEX colors into InVideo immediately. This ensures every single output is 100% compliant with your brand guidelines.

Step 2: Integrating Visual Assets (Media Syncing)

This is where many novices fail. Instead of dropping clips randomly, treat InVideo as a sync tool.

  1. Timeline Setup: Import your entire script text first. This provides the rough timing guide.
  2. Source Material Insertion: Drop your corresponding screen recordings and narrated audio track onto the timeline.
  3. Visual Fill: Only where the video requires a visual pause, B-roll footage, or animated text, do you use the InVideo library. Use keywords derived directly from your script ("API integration fail," "dashboard analytics chart").

Example Case Study: When teaching about the 'multi-tiered pricing model,' a static screenshot is insufficient. Use the 'Animated Diagram' feature in the library to show tiers expanding or collapsing, visually demonstrating the escalation of value.

Step 3: Crafting High-Conversion Text Overlays

People watch tutorials without sound. Your text overlays are your lifeline.

  • Keep it Concise: Never display a full sentence. Use bullet points and headline fragments (e.g., Instead of: "The core feature is that it allows for rapid deployment," use: ✓ Rapid Deployment).
  • Hierarchy Matters: Use strong tags for keywords (e.g., KEY ACTION) and subtle emphasis for context (e.g., *Optimal setup requires*). Don't overuse animation on text—it causes cognitive overload.
  • Consistency: Use the same animation timing and style for every major point in your tutorial. This builds unconscious familiarity and trust with the viewer.
